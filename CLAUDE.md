## Project Overview

**Sargassoti** — A Flutter app for reporting beach conditions in Guadeloupe. Users report sargasses, waves, crowd, and noise levels (0–3 scale) with a rating and optional comment. The app is in French (Creole name, French UI).

## Commands

```bash
flutter pub get              # Install dependencies
flutter run                  # Run the app (requires Firebase config)
flutter analyze              # Lint / static analysis
flutter test                 # Run tests (currently only a default widget test)
flutter test test/widget_test.dart  # Run single test file
```

Firebase must be configured before `flutter run` works. See README.md for setup steps.

## Architecture

```
lib/
  main.dart                    # Entry point: Firebase init, anonymous auth, Provider setup
  theme.dart                   # AppTheme with light/dark ThemeData
  models/beach_report.dart     # BeachReport model with toJson/fromJson
  providers/theme_provider.dart # ChangeNotifier for theme (system/light/dark), persisted via SharedPreferences
  services/firestore_service.dart # Firestore CRUD: addReport, getReports (stream), canAddReport, getBeachNames, getBeachPopularity
  screens/
    splash_screen.dart         # 2-second splash → HomeScreen
    home_screen.dart           # Main screen: beach list with filters, sorted by report count
    report_screen.dart         # Submit a report: beach selector, level squares (2x2 grid), star rating, comment
    comments_screen.dart       # View all comments for a beach
    settings_screen.dart       # Theme picker (system/light/dark)
  widgets/beach_card.dart     # Card displaying beach name, latest report, level icons, WhatsApp share
```

**Navigation flow**: SplashScreen → HomeScreen (with Drawer → SettingsScreen). Tapping a BeachCard → ReportScreen. ReportScreen links to CommentsScreen.

## Key Design Decisions

- **Beaches are not hardcoded in the app** — they emerge from Firestore reports. The seed script at `bin/firestore.seed.js` populates initial beach data using `firebase-admin`.
- **Beach IDs** are derived from beach names: `beachName.toLowerCase().replaceAll(' ', '-')`. A custom `StringNormalize` extension in `beach_card.dart` strips French diacritics for image asset lookups.
- **Beach images** are local assets at `assets/images/{beachId}-min.jpg`. The image filename is constructed by normalizing the `beachId` via NFD decomposition + diacritic removal.
- **Anti-spam**: `FirestoreService.canAddReport()` checks if a user's last report is ≥2 hours old (note: current implementation uses `diff.inHours >= 0`, which is effectively no rate limit — likely should be `>= 2`).
- **WhatsApp share** uses `url_launcher` to open `wa.me` with a pre-formatted message.
- **Theme persistence** uses `SharedPreferences` with string keys (`'system'`/`'light'`/`'dark'`).

## Firebase

- Project ID: `sablessargasses`
- Auth: Anonymous only (no user registration)
- Single Firestore collection: `reports` — documents contain beachId, beachName, sargassesLevel, wavesLevel, crowdLevel, noiseLevel, rating, comment, userId, timestamp
- `firebase_options.dart` is not in version control — generated by `flutterfire configure`
- Android package: `com.loicdurand.sargassoti`

## Seed Script (`bin/firestore.seed.js`)

Node.js script using `firebase-admin`. Requires `serviceAccountKey.json` in `bin/`. Run with `node bin/firestore.seed.js`. It deletes all existing reports then inserts 15 predefined beaches.
